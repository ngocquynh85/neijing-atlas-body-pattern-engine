# Token Justification

## Summary

NEIJING ATLAS is token-intensive because it combines classical text reasoning, source-grounded concept mapping, free-form journal interpretation, multilingual explanation, and safety review.

It is not a single chatbot prompt. It is a versioned AI workflow that turns daily body observations into non-diagnostic traditional pattern lenses grounded in classical concepts and reviewed for safety.

## Workload dimensions

### 1. Classical corpus processing

- passage segmentation
- variant/version alignment
- concept extraction
- terminology normalization
- translation comparison
- commentary summarization
- concept graph construction

### 2. Modern journal processing

- free-form signal extraction
- structured check-in normalization
- body-signal taxonomy mapping
- pattern lens scoring
- red-flag detection
- reflection plan generation

### 3. Long-context synthesis

- weekly correlation reports
- repeated pattern detection
- cross-day context windows
- consistency with previous explanations
- user-specific observation history

### 4. Multilingual output

- Classical Chinese source passages
- Modern Chinese explanation
- English explanation
- Vietnamese explanation
- terminology glossary across languages

### 5. Safety review

- no diagnosis
- no treatment claims
- no herb/prescription instructions
- emergency red-flag routing
- uncertainty and confidence notes

## Planning ranges

These are planning estimates, not claimed usage.

| Phase | Purpose | Estimated range |
|---|---|---:|
| Pilot | Validate signal extraction, concept mapping, safety, and weekly report generation | 1B–5B tokens |
| Canon graph build | Process texts, translations, concepts, commentary, and glossary variants | 10B–50B tokens |
| User-pattern pilot | Daily entries, weekly reports, safety review, multilingual outputs | 50B–200B tokens |
| Quality improvement | Reprocess ambiguous mappings, compare prompts, reviewer audits | 200B–800B tokens |
| Research mode | Long-context comparison across canon, commentary, and longitudinal journals | 500B–2T+ tokens |

## Why large quota is useful

A large quota allows the project to move beyond a static demo and test the real workload:

- repeated daily and weekly processing
- long-context correlation summaries
- multilingual explanation consistency
- prompt/version comparisons
- safety classifier improvements
- source-grounded concept graph refinement

The system can start with a small mock/pilot flow and scale naturally as licensed sources and user-study journals are added.
